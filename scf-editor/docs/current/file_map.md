# SCF Editor — File Map

Concise index of every file in the project, grouped by area. Use this to decide which files to request when starting work on a task.

---

## Backend — Core schema & storage

- **`entity_registry.py`** — Single source of truth for all ~96 entity types. Defines `EntityDef` / `FieldDef`, auto-injects `lifecycle_status` / version-chain fields / `external_id` fields based on entity flags. Touch this file to add, rename, or restructure entities. Almost every other backend file imports from it.
- **`database.py`** — SQLite layer. Auto-creates tables from the registry, runs simple `ALTER TABLE ADD COLUMN` migrations, exposes generic CRUD (`create_entity`, `update_entity`, `list_entities`, etc.), plus batched helpers (`batch_entity_counts`, `list_entities_batch`) used by the tree loader. Also handles project creation and the projects directory.
- **`queries.py`** — Predefined queries used by the Query Explorer page: `character_journey`, `location_breakdown`, `scene_context`, `character_crossover`, `project_stats`. Raw SQL with JOINs across junction tables.

## Backend — Web app & routing

- **`main.py`** — The FastAPI app. Project management routes (create/import/open/close/download), the `/browse` and `/query` pages, generic CRUD API, search, the inline-link API for relationship panels, the `display_label` Jinja filter that renders lowercase enum values as human-readable labels. Mounts the screenplay and images routers.
- **`screenplay_api.py`** — FastAPI router mounted at `/api/screenplay-v2/*`. Load/save the full screenplay, navigator data (scenes/characters/locations/props), autocomplete endpoints, prop tag CRUD, version management (publish/list/restore/delete), Fountain import, blank-screenplay scaffold.
- **`images_api.py`** — FastAPI router for reference images. Upload/list/update-description/delete. Files stored under `projects/<name>/sourcefiles/images/<entity_type>/<entity_id>/`. Includes a static-file serving route for `/project-files/...`. Supported entities listed in `IMAGE_ENTITY_TYPES`.

## Backend — Screenplay subsystem

Two parallel paths exist here; both need attention when changing screenplay behavior.

- **`screenplay_db.py`** — The v2 SQLite-native screenplay engine. Manages `screenplay_lines`, `screenplay_title_page`, `screenplay_versions` (and snapshot tables), `screenplay_prop_tags`. Functions: `load_screenplay`, `save_screenplay`, `import_fountain_to_lines` (the v2 import path), `parse_heading`, `_rebuild_junctions`, `publish_version` / `restore_version` / `list_versions`, prop-tag CRUD. Phase 1C lowercased the enum mappings here.
- **`fountain_parser.py`** — Pure Fountain text parser. Extracts scenes, characters (with first-appearance descriptions), locations, props (whitelist-based), and scene-character links from Fountain source. No DB writes. Heavy on heuristics: encoding normalization, character-cue detection, compound-character dedup, prop confidence scoring.
- **`fountain_import.py`** — The older orchestrator that takes parsed `FountainData` and writes entities via `database.py` CRUD. Used for *new-project* fountain imports from the landing page. Distinct from the v2 path in `screenplay_db.import_fountain_to_lines` which writes directly to `screenplay_lines`. Imports a `fountain_anchors` module (not shown in this snapshot) for anchor injection into the working text copy.

## Documentation

- **`docs/current/conventions.md`** — Canonical authority on cross-cutting rules: versioning model, status taxonomy (lifecycle / production / writing-process / entity-specific), casing convention, reference field semantics, OMC posture, the bundle pattern, naming conventions. The doc generator reads this file and embeds it into both generated docs.
- **`docs/current/format_overview.md`** — Conceptual intro to SCF: what it is, why it exists, design principles, relationships to USD / SSF / Fountain / OMC. Hand-authored.
- **`docs/ai_context/schema_snapshot.md`** — Auto-generated compact reference for AI session priming. Lists every entity by tier with one-line descriptions, schema-flag counts, and a full entity reference graph. Regenerated by `scripts/generate_schema_docs.py`.
- **`docs/current/schema_reference.md`** — Auto-generated comprehensive reference: every entity, every field, every option, full metadata. Not included in this snapshot but produced by the same generator script.

## Scripts

- **`scripts/generate_schema_docs.py`** — Reads `entity_registry.py` + `conventions.md` and writes `schema_reference.md` and `schema_snapshot.md`. Run after schema changes.
- **`scripts/scripts_README.md`** — Usage notes for the doc generator (PowerShell-flavored).

## Frontend — Templates (Jinja2)

- **`templates/index.html`** — Landing page / project selector. New project, import `.scf`, import `.fountain` (with progress overlay), open existing, download. The Fountain import flow does a two-step create-then-import via the v2 API.
- **`templates/browse.html`** — Main two-panel browse view. Header with global search and nav links, tree on left, entity editor on right.
- **`templates/query.html`** — Query Explorer page with five query cards.
- **`templates/screenplay.html`** — Screenplay editor page. Navigator + CodeMirror panel + status bar + version drawer + help modal. Two states: empty (import/create options) vs populated (full editor).
- **`templates/partials/entity_tree.html`** — Left-panel entity tree: category → entity-type group → item list. Includes collapse-all / expand-all toolbar and per-group + create buttons.
- **`templates/partials/entity_form.html`** — The big one. Renders the entity editor: header, tab bar, field-rendering loop (one branch per field type), save bar, inline relationship link panels for scenes/sequences/acts, reverse-link "Appears In" sections, and the Reference tab for image-enabled entities.
- **`templates/partials/link_panel.html`** — Reusable inline relationship-linking widget. Takes parameters like `junction_type`, `meta_field`, `meta_options`, and renders an autocomplete + chip list.
- **`templates/partials/reverse_links.html`** — Read-only "Appears In" section for showing reverse FK relationships (character → scenes, scene → story beats, etc.).
- **`templates/partials/search_results.html`** — Dropdown items for the global search box.

## Frontend — CSS

- **`static/css/style.css`** — Main stylesheet. Design tokens, landing page, browse layout (header / tree / editor), forms, buttons, Query Explorer, inline link panels, "Appears In" sections, tree controls. Touch this for anything outside the screenplay editor.
- **`static/css/screenplay.css`** — Screenplay editor styles: navigator panel, status bar, CodeMirror line decorations (`cm-scf-heading`, `cm-scf-character`, etc.), structural gap classes (`cm-scf-gap-scene`, etc.), active-line mode overrides, autocomplete dropdown, help modal, version drawer.
- **`static/css/screenplay_props.css`** — Prop highlight underline style and the prop-tag popover.
- **`static/css/screenplay_versions.css`** — Version drawer styles. **Note:** content here appears duplicated in `screenplay.css`; one of the two is likely vestigial.
- **`static/css/entity_images.css`** — Reference tab: upload zone, image grid, image cards, lightbox.
- **`static/css/entity_tooltips.css`** — JS-positioned hover tooltips for entity tree group headers (description text on hover).

## Frontend — JavaScript

- **`static/js/app.js`** — Browse-page client. Tree collapse/category state in `localStorage`, panel resize with persistence, sort toggles (alpha vs creation order), global search wiring, keyboard shortcuts (`Cmd/Ctrl+K`), inline link panel logic (autocomplete + chip CRUD), htmx event hooks for save indicator and editor state preservation (active tab + scroll). Touch this for browse-view interactions.
- **`static/js/query.js`** — Query Explorer client. Runs each predefined query and renders tables/cards into `#query-results`.
- **`static/js/screenplay-editor.js`** — The big one. ES module, CodeMirror 6 editor. Architecture: a `StateField` holds sticky per-line types (no runtime Fountain reclassification); types are only changed by explicit user actions (Tab, Enter, Backspace merge, autocomplete, paste). Document contains only real content lines — visual spacing is CSS via gap classes. Mode system (description / scene / character / dialogue / transition) with auto-switching on Enter. Includes autocomplete (characters / locations / prefixes / time-of-day), entity ID caching, prop tagging (Ctrl+P), navigator scene/character/location rendering with filter highlighting, copy/paste preserving line types via a custom MIME, load/save with full reload to pick up server-resolved IDs.
- **`static/js/screenplay-versions.js`** — Version drawer logic: open/close, publish, fetch + render version list, restore, delete.
- **`static/js/entity-images.js`** — Reference tab client: drag-and-drop upload, thumbnail grid, debounced description save, delete confirmation, keyboard-navigable lightbox.
- **`static/js/entity-tooltips.js`** — Tree group header tooltips with 450ms hover delay, body-mounted to escape `overflow: hidden`.

## Quick lookup — "where does X live?"

| If the task touches… | Start with |
|---|---|
| Adding or renaming an entity | `entity_registry.py`, then regenerate docs |
| A schema convention (status, versioning, casing) | `conventions.md`, then `entity_registry.py` |
| Generic CRUD or DB performance | `database.py` |
| The browse/tree view | `main.py` (`/browse`, `/htmx/tree`), `entity_tree.html`, `entity_form.html`, `app.js`, `style.css` |
| The entity editor form | `entity_form.html`, `style.css`, `app.js` |
| Inline relationship linking | `link_panel.html`, `_LINK_CONFIG` in `main.py`, link-panel section in `app.js` |
| Reverse "Appears In" rendering | `reverse_links.html`, `_query_links` in `main.py` |
| Reference images | `images_api.py`, `entity_images.css`, `entity-images.js`, `entity_form.html` (Reference tab) |
| Query Explorer | `queries.py`, `main.py` (query routes), `query.html`, `query.js` |
| The Screenplay editor itself | `screenplay-editor.js` (architecture lives here), `screenplay.css`, `screenplay.html`, `screenplay_db.py`, `screenplay_api.py` |
| Fountain import (new project flow) | `fountain_parser.py`, `fountain_import.py`, `index.html` |
| Fountain import (into existing project) | `fountain_parser.py`, `screenplay_db.import_fountain_to_lines`, `screenplay_api.py` |
| Screenplay versioning | `screenplay_db.py` (publish/restore/list), `screenplay_api.py`, `screenplay-versions.js`, version drawer CSS |
| Prop tagging | `screenplay_db.py` (prop tag funcs), `screenplay_api.py`, `screenplay-editor.js` (prop popover), `screenplay_props.css` |
| Auto-generated docs | `scripts/generate_schema_docs.py`, `conventions.md`, `entity_registry.py` |
