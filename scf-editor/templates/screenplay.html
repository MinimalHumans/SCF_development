Screenplay Editor — Quick Reference

Modes

The editor uses a mode system to format your text. The current mode is shown in the status bar at the bottom left.

Press Tab to cycle forward through modes: Description → Scene Heading → Character → Dialogue → Transition

Press Shift+Tab to cycle backward.

Writing in Each Mode

Description
Standard action lines. Left-aligned, white text. Use for scene descriptions, stage directions, and narrative.
Scene Heading
Creates a new scene and location. Text is auto-capitalized with a highlighted background. Standard format is INT. LOCATION - DAY but any text works.
Character
Character name before dialogue. Auto-capitalized, centered. Autocomplete suggests existing characters as you type.
Dialogue
What the character says. Indented with a light purple tint.
Transition
Right-aligned, auto-capitalized. For CUT TO:, FADE OUT, etc.

Structural Spacing

The editor automatically adds visual spacing between different screenplay elements — you don't need to insert blank lines between scenes, before character cues, or after dialogue blocks. The spacing is handled by the editor's formatting.

If you want an intentional extra blank line (e.g., a dramatic pause within action text), press Enter on an empty line.

Automatic Mode Switching

The editor switches mode automatically in these cases:

  • Enter after a Scene Heading → switches to Description
  • Enter after a Character name → switches to Dialogue
  • Enter after Dialogue → switches to Description

Autocomplete

  • In Character mode, type 2+ characters to see matching character names from the database.
  • In Scene Heading mode, after typing INT. or EXT. followed by text, matching locations appear.
  • Select a suggestion with Enter or Tab.
  • New names are created as entities on save.

Saving

Press Ctrl+S (or Cmd+S on Mac) or click Save. The save process creates any new characters, locations, or scenes and rebuilds scene-character relationships.
