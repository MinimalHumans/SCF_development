Screenplay Editor — Quick Reference

Modes

The editor uses a mode system to format your text. The current mode is shown in the status bar at the bottom left.

Press Tab to cycle forward through modes: Description → Scene Heading → Character → Dialogue → Transition

Press Shift+Tab to cycle backward.

Writing in Each Mode

Description
Standard action lines. Left-aligned, white text. Use for scene descriptions, stage directions, and narrative.
Scene Heading
Creates a new scene and location. Text is auto-capitalized with a highlighted background. Standard format is INT. LOCATION - DAY but any text works — the location is created from whatever you type.
Character
Character name before dialogue. Auto-capitalized, centered. Autocomplete suggests existing characters as you type. New names are created as character entities on save.
Dialogue
What the character says. Indented with a light purple tint.
Transition
Right-aligned, auto-capitalized. For CUT TO:, FADE OUT, etc.

Automatic Mode Switching

The editor switches mode automatically in these cases:

  • Enter after a Scene Heading → switches to Description
  • Enter after a Character name → switches to Dialogue
  • Enter after Dialogue → switches to Description

You can always override by pressing Tab to pick a different mode. The auto-switch only happens on Enter, never from clicking or arrow keys.

Clicking Existing Lines

When you click on an existing line, the mode automatically switches to match that line's type. This prevents accidentally mixing formats on a single line.

Clicking on a blank line keeps whatever mode you're currently in.

Scene Headings & Locations

Every scene heading creates a location entity in the SCF database. The location name is extracted by stripping the INT/EXT prefix and time-of-day suffix.

Supported prefixes: INT. EXT. INT./EXT. I/E. EST.

Supported time suffixes: DAY NIGHT MORNING AFTERNOON EVENING DAWN DUSK SUNSET SUNRISE CONTINUOUS SAME TIME LATER MOMENTS LATER

Headings without a prefix are also valid — the full text becomes the location name.

Multiple headings sharing the same location name (e.g. INT. WAREHOUSE - DAY and EXT. WAREHOUSE - NIGHT) link to the same location entity.

Autocomplete

  • In Character mode, type 2+ characters to see matching character names from the database.
  • In Scene Heading mode, after typing INT. or EXT. followed by text, matching locations appear.
  • Select a suggestion with Enter or Tab, or use arrow keys to navigate the dropdown.
  • Typing a name that doesn't exist yet will create it as a new entity when you save.

Saving

Press Ctrl+S (or Cmd+S on Mac) or click the Save button. The save process:

  1. Writes all lines to the database with their types and entity links
  2. Creates any new characters, locations, or scenes that don't exist yet
  3. Rebuilds scene-character relationships from the screenplay content
  4. Updates the navigator panel with fresh data

The status bar shows save state: Ready, Unsaved •, Saving…, or Saved ✓.

Navigator Panel

The left panel shows scenes, characters, and locations extracted from the screenplay.

  • Click a scene to jump to that heading in the editor.
  • Click a character or location to filter the scene list to only scenes containing that entity.
  • Click the → arrow next to any item to open it in the Entity Browser for detailed editing.
